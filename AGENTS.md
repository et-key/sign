# Agent Instructions for Sign Workspace

## Sign Language Philosophy & Core Mechanics

1. **未定義識別子の「無（Unit）」への収束**
   - Signには `null`, `nil`, `false` などの予約語や特別なプリミティブ型は存在しない。
   - **「定義されていない識別子はすべて `__unit`（空の余積・無）として評価される」**という強力な性質を持つ。
   - これにより、`@lazy tick` や `iterator next` のように、任意の英単語やUnicode記号（`❓`、`❌`など）を「予約語ゼロ・コストゼロの仮想キーワード」として自由に用いて、自然言語的・DSL的なコードを記述できる。

2. **UnitとFalseの同型性 (Isomorphism) と短絡評価**
   - 論理学における「偽（False）」は「無（Unit）」と同型である。
   - 比較演算等で条件が満たされない場合は `__unit` が返る。
   - 言語組み込みの論理演算子（`&`, `|`）は短絡評価（Short-circuit）を行う。左辺が `__unit`（偽）の場合、`&` は即座に `__unit` を返し、`|` は右辺を評価する。
   - これらとUnit伝播を組み合わせることで、`if` 文や `match_case` などの制御構文すら不要となり、`xs == __ & @x y ys | x (append xs (y ys))` のように、複雑な条件分岐や再帰の終了条件を「純粋な数式（論理演算）」として簡潔に表現できる。

3. **予約語の排除とミニマリズム**
   - 構文そのものから不要な制御構文や予約語を削ぎ落とし、純粋な計算モデル（関数の適用と結合）に還元する。
   - メタ操作（例: `#`, `##` によるスコープ外アクセス）やAST操作以外の場面では、極力、関数や暗黙の言語仕様（Unitの性質など）の組み合わせで解決する。
