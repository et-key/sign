# Sign 言語仕様・設計ノート：関数の定義と評価におけるモナドとコモナドに類似する特徴

## Maybe（一般言語でいう `if~then~else` が該当）
   - 引数に暗黙に `Unit` が渡されたとき、関数は実行されない。（※ `Unit` を明示で渡す場合はプレースホルダなので別の意味になる）
   - よって、`else` 相当はdefault引数となる（defaultを明示しないときは `Unit` がdefaultとなる）。
   - ※ 引数不足だと、引数がある部分を関数に宛がった上での関数を返す → 実行されない関数は `Unit` を返すので、`Unit` を渡し `Unit` を返す…という動作。
   - ※ Sign上では **未実行関数 ＝ Unitと同型**。

```sign

f : x y ? x + y

`3 < 2 は偽なので Unitが返る
f 3 < 2 2

g :
		x : 0
   	y : 0
   ?
		x + y

`デフォルト値が`0`なので、`3 < 2`で`Unit`が返っても`0`が代入され、2となる。
g 3 < 2 2

```

## List（一般言語でいうイテレータが該当）
  - 関数にListを渡すときは必ず後置`~`を使う
  - 後置 `~` を使ったときに、それぞれの引数リストに分配して渡される

```sign

f : a b c ? a + b + c

list1 : 1 2 3
list2 : 1 , 2 , 3

`f 1 2 3と同じ意味
f list1~
f list2~
f [1,2,3]~
f [1 2 3]~

```

## Either（一般言語でいう `try~catch~finally` が該当）
   - `Match_Case` による関数内での条件分岐を記述することができる → 関数の無限ループを防ぐことになる。
   - `Match_Case` 内に異常条件がある時、`Unit` を返せばよい（assertion: エラー表示し `Unit` を返す）。
   - であれば、正常終了や正常ループを書ける。

```sign

f : x y ?
	x < 0 : `Error : x is negative`
	y < 0 : `Error : y is negative`
	x * y

`以下は6を返す。
f 2 3

`以下は`Error : x is negative`を返す
f -1 -1
f -1 2

`以下は`Error : y is negative`を返す
f 2 -1

```

## State（一般言語でいう「変数の定義と変更」が該当）
   - 引数リストにdefault値を設定する際、状態を扱う簡易計算を許容する。

```sign

f :
		x : x + 1
		y : x + y
	?
		x > 10 : y
		f x y

`以下は60を返す。
f 0 5

```

## IO Reader Writer
   - default値の計算内に前置 `@`、を含む式を使ってもよい。
   - 関数本体には、同様に前置 `@`、中置 `#` を含む式を使ってもよい。　
   - 関数定義内で 中置`#` の返値はアドレスなので、呼び出し元に前置`@`を書いてIOを渡すことができる（Reader/Writer関手の合成）。
   - 単にアドレスだけ渡したいときは、前置`$`付きの式を渡す。

```sign

F : x y ? x # @y
G : f y ? @x y

`IOや高階関数でない場合は、冗長にアドレスが必要となるなどパフォーマンスには影響がある
`記述や扱いには慎重を要するが、その分表現の幅は広がる。

`書き込み、読み込みのちに 5 が返る
G @[F 0x10000 $[+ 2]] 3

```

## Zipper
   - 引数リストに残余引数リスト演算子の前置 `~` を使うことができる。（※リフト(lifting)。w a → w(w a) の操作）
   - 残余引数リストに渡された値は「リスト」に持ち上がるため、内側で通常の`get`が可能。

```sign

f : ~a ? a ' 0 + a ' 2 + a ' 3

` 1 2 3がリストとしてfに渡される
f 1 2 3

```

   - 関数適用時、後置 `~` を使うと双代数になるため、前置`~`を使いリストの高階操作ができる。

```sign

map : f x ~y ? @f x , map f y~

map $[* 2] 1 2 3 4 5

```

## Store
   - もし仮引数の名前が、渡された辞書のkey名と一致するなら仮引数にそれを代入する。
   - 構造体というポインタへの参照を渡すため、仮引数部は必ず何らかのカッコで囲む。

```sign

get_age : [age ~obj] ? age

dict :
   name : `Johnny`
   age  : 20


` name ' key の操作がされる。
get_age dict

```
