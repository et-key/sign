# Sign 文字列リテラルとコメント構文

## 概要

Sign の文字列は必ずバッククォート（`` ` ``）で囲む。
コメントと文字列の区別は**先読みルール（閉じバッククォートの有無）**によって決まる。

---

## 1. 文字列リテラル

### 定義

```
string = ` [^\n\r`]* `
```

開始と終了をバッククォートで挟んだ、改行（`\n`/`\r`）とバッククォート自身を除く任意の文字列。

```sign
greeting : `Hello, World!`   ` greeting に文字列を束縛
message  : `Sign is elegant`
```

### 特性

- 文字列はシングルライン（改行を含まない）
- 空文字列は `__`（Unit）と同型
- 文字列中にバッククォートは含められない（コメント開始と区別できないため）

---

## 2. コメント構文

### 定義（先読みルール）

```
comment = SOL ` [^\n\r`]* EOL        ← 行頭 ` かつ閉じ ` がない → コメント
string  =     ` [^\n\r`]* `          ← 閉じ ` がある → 文字列リテラル
```

> [!IMPORTANT]
> **先読みルール**：バッククォートを見つけたら、同じ行に閉じバッククォートがあるか確認する。
> - **閉じあり** → 文字列リテラル `` `[^\n\r`]*` ``
> - **閉じなし** → コメント（EOL まで）
>
> これにより予約語ゼロかつ曖昧性ゼロで文字列とコメントを区別できる。

```sign
`これはコメント（閉じなし → EOLまでコメント）
`main.sn`@~    ← 閉じあり → `main.sn` は文字列リテラル（インポートパス）
x : `Hello`    ← 閉じあり → 文字列リテラル
```

### 衝突がない理由

「行頭に文字列が来ることは構文上ない」という前提のもと、
行頭のバッククォートは「コメントか文字列か」を閉じバッククォートの有無だけで区別できる：

```sign
`path/to/file`@~    ← 閉じあり → 文字列（インポートパス）
` これはコメント     ← 閉じなし → コメント
```

---

## 3. インデント内のバッククォート（ドキュメント文字列）

インデント（タブ）の後に `` ` `` が来る場合は、行頭ではないのでコメントにならない。
先読みルール（閉じあり=文字列、閉じなし=コメント）が適用される。

```sign
calc_func : x ?
\t`ドキュメント文字列（タブ後・閉じあり → 文字列として評価されるが捨てられる）`
\tx * 2
```

この文字列は評価されますが戻り値として使われないため、事実上コメントとして機能します。

---

## 4. 最終行の返値

Sign の実行モデルでは**すべてのファイルは関数**である（`impl/core/execution_model.md` 参照）。

したがって、**ファイルの最終行（最後の非コメント式）がそのファイルの返値**になる。

```sign
` greet.sn
name : `World`
`Hello, {name}!`    ← 最終行：このファイルの返値
```

```sign
` calc.sn
add : x y ? x + y
mul : x y ? x * y

add 3 4    ← 最終行：7 がこのファイルの返値
```

---

## 5. 複数行の文字列

Sign の文字列は**シングルライン**が基本。

複数行の文字列を扱いたい場合は、文字列のリストまたは連接演算子（空白）を使う：

```sign
` 複数文字列の組み立て
lines : `Line 1` `Line 2` `Line 3`   ` リスト構築

`改行文字自体を文字として扱う
text  : `Line 1` \
 `Line 2`
```

注意点として、この言語にはエスケープシーケンスを用意する必要が無い。
`\`後の任意の1文字が文字として扱われる。

---

## 6. Unicode との関係

文字列は `0u` リテラル（Unicode コードポイント）のシーケンスとして扱える：

```sign
` ASCII 文字列
hello : `Hello`

` 個別コードポイントとしてアクセス
h : `hello` ' 0   ` → \h（0u48）
```

---

## 7. 文法仕様（実装者向けメモ）

- 字句解析レベルでコメント（閉じなしバッククォート行）を削除しても構わない
- シンタックスハイライトや Language Server は**「行頭から始まり閉じバッククォートがない `` ` `` 行」**をコメント色で表示することが推奨される
- PEG では先読み（lookahead）で閉じバッククォートの有無を判定する
- 詳細な実装仕様は `impl/syntax/grammar.pegjs` を参照

---

## 設計の意図

- **バッククォートのみで文字列とコメントを兼用** → 予約語や専用引用符を増やさず、記号を最小限に保てる
- **行スコープ** → コンパイラがトークン境界を一行ずつ静的に解析できる
- **コメントは SOL かつ閉じバッククォートなし** → 視覚的に明確、予約語不要
- **最終行返値** → 「ファイル = 関数」の実行モデルと一致
