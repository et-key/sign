# Sign言語の論理値設計：圏論的観点から記述する仕様

## 1. 概要

Sign言語における論理値の扱いについて、数学的基盤と言語設計の一貫性を高めるための仕様を示します。

- Unit(`__`) : 空のリストと未評価の関数リテラルがfalse
- その他すべての値（0を含む）はtrue

## 2. 理論的根拠

### 2.1 圏論的な視点

圏論において、0は「初期対象」として重要な役割を持ちます。初期対象は任意の対象への一意な射を持つ特殊な存在であり、「無」ではなく「何か」（機能を持つもの）として扱われます。

一方、Unit/空のリストは「何もない」状態を表現するため、論理的な「偽」として扱うのが自然です。これはモナド理論におけるNothingの概念とも一致します。

### 2.2 数学的一貫性

ZFC集合論では0はRing（環）における加法単位元として特別な役割を持ちます。一方で圏論的には0は初期対象として機能します。Sign言語の「見えない強い型付け」や「リストを基盤とした統一データモデル」といった特性は、圏論的に記述したとき整合するため、0をtrueとして扱うことでより一貫した数学的根拠が得られます。

## 3. 実用的メリット

### 3.1 明確なセマンティクス

- 「値が存在するか」と「値が0か」の区別が明確になります
- 数値計算の結果としての0と論理的な「偽」の概念が明確に分離されます
- 数学的表現との親和性が高まります（数学では0も有効な数として扱われるため）

### 3.2 言語設計の一貫性

Sign言語の「リストを基盤とした統一データモデル」において、リストの「空」状態をfalseとすることで、データモデルと論理値の関係が明確になります。数値は別の概念として扱われるべきです。

## 4. 具体例

### 論理演算の例

```sign

` 0はtrueのため、xを返す
0 & x

` 空リストはfalseのため、Unit(`__`)を返す
[] & x

` 1はtrueのため、xを返す
1 & x
```

### 比較演算との例

```sign

` 比較がtrueなら、左辺を返す。
` 例示の場合は 2 となる
2 < 3

` 比較演算は三項までを許容する。比較がtrueなら中央を返す
` 例示の場合は 3 となる
1 < 3 < 5

` 比較演算が失敗すればUnit(`__`)を返す。
3 < 2   ` → __

```

### 部分適用と、評価除外の例

```sign

` 関数定義
f : x y ? x + y

` 明示的なHole（_）は部分適用とする
f _ 4 == [+ 4]

` Unitになる結果を渡した場合は短絡でUnitに評価してよい
f 3 > 2 4

```

## 5. 結論

Unit(`__`)/空のリストをfalseとし、0を含むその他の値をtrueとする設計は、
圏論的に記述した際の一貫性を持ち、
より直感的で明確なセマンティクスを提供します。この仕様によって、Sign言語はより表現力豊かで数学的に整合性の高い言語として機能します。

特に重要なのは、数値の0と論理的な「無」を明確に区別することで、プログラマが直感的に理解しやすい一貫した論理体系を提供することです。

## 6. `!` 演算子の意味論：SKI コンビネータによる根拠

Sign の真偽値は **SKI コンビネータ** の構造に根拠を持つ。

| SKI | λ式 | Sign での対応 | 意味 |
|:---:|:---:|:---:|---|
| **K**（左返し） | `λx.λy.x` | **Id射**（`[__ ]` / `[ __]`） | **真（True）** |
| **K\***（右返し） | `λx.λy.y` | **`__`（Unit）** | **偽（False）** |

- **偽（`__`）** = void 関数（引数を吸収して何も返さない K* コンビネータ）
- **真** = Id 射（引数をそのまま返す恒等射 K コンビネータ）

これにより、`!`（前置 NOT）は以下のように定義される：

```sign
!__    ` → Id射（非Unit・真に相当）  ← False → True
!expr  ` → __                        ← True  → False（expr が非Unit の場合）
```

> [!IMPORTANT]
> **`!__` は具体的な数値（`1` 等）を返さない。**
>
> `!__` に `1` を割り当てると Boolean 型を暗黙に再導入することになり、
> Sign の設計原則（Boolean 型不要）と矛盾する。
> `!__` の結果は「Unit でない何か（Id射）」であり、
> `&`/`|` チェーンの中で真として機能することが保証される。

### 比較演算との対応

```sign
` 2 < 3 は真 → 左辺値 2 を返す（Id射として機能）
2 < 3   ` → 2

` 3 < 2 は偽 → __ を返す（void/K* として機能）
3 < 2   ` → __

` 結果を ! で反転
!(2 < 3) ` → __    ← 真 → 偽
!(3 < 2) ` → Id射  ← 偽 → 真
```

## 7. `==`（構造比較）の意味論

`==` は算術比較（`=`）とは異なる**構造的同一性**を判定する演算子。

### 7.1 判定基準

- **同型（Isomorphic）かつ同値（Equal）** であることが条件
- 再帰的に構造を比較する（深さ制限なし）
- `=`（算術比較）は `Int` / `Float` / 文字型 / Unicode型 / アドレス型のみ対象だが、
  `==` はリストや辞書も含む任意の構造に適用可能

### 7.2 `=` との使い分け

| 演算子 | 対象 | リスト直接比較 |
|:---:|---|:---:|
| `=` | Number系Atom・文字型・Unicode型・アドレス型 | ❌（型エラー） |
| `==` | 任意の構造（Atom・List・Struct・Dict） | ✅ |

```sign
` 算術比較（= ）
3 = 3       ` → 3   （等値・真）
3 = 4       ` → __  （不等・偽）

` 構造比較（==）
[1 2 3] == [1 2 3]  ` → [1 2 3]  （同型同値・真）
[1 2 3] == [1 2 4]  ` → __       （値が異なる・偽）
[1 2]   == [1 2 3]  ` → __       （構造が異なる・偽）
```

### 7.3 循環参照を含む構造の `==`

循環参照を含む構造に `==` を適用した場合、**`__`（Unit）を返す**。
無限ループには陥らない。

```sign
` 循環参照を含む構造（実装上の注意）
` → 比較器が循環を検出した時点で __ を返す
```

> [!NOTE]
> 循環参照の検出は、訪問済みノードのアドレスを追跡することで実装する。
> 同一アドレスへの再訪問を検出した場合、そのサブツリーの比較結果を `__` とする。
