# 関数定義とその型について

Signでは、関数定義の方法が多岐にわたって存在する。  
よって、これら関数定義の方法について、具体例を交えたドキュメントをここに記す。

## ポイントフリー記述

全ての演算子を関数として扱う場合、任意のカッコで演算子を囲むことで関数として扱う。  
この時、前置演算子は、`[<op>_]` 後置演算子は `[_<op>]`と記述する。  
ポイントフリー記述の二項演算子は、複数の引数を貪欲に演算する。  
単項式の後ろに`,`を付けたポイントフリー記述は、そのすべてに適用される。

```Sign

` Sumと同等
[+] 1 2 3 4 5

` mapと同等
[* 2,] 1 2 3 4 5

` 否定する関数
[!_] 2 < 3

` 階乗する関数
[_!] 5
```

## ? 演算子を用いた定義方法

`?`演算子は、左辺に引数リスト、右辺に本体を有する演算子と解釈できる。

```Sign

f : x y ? x + y
```

### match_caseの記述

`?` の右辺を改行・インデントブロックを挟むことで、本体内の : 演算子はmatch_caseとなる。

```Sign

f : x y ?
	x > 3 : x - y
	y < 3 : x + y
	x y
```

### デフォルト引数の記述

`?` の左辺インデントブロックは : 演算子がデフォルト引数の記述となる。

```Sign

f :
		x
		y : x + 1
		z : y + 1
	? x y z
```

### デフォルト引数と、match_caseの記述は共存可能。

```Sign

f :
		x
		y : x + 1
		z : y + 1
	? x y z
		x > 3 : x - y
		y < 3 : x + y
		z
```

これらの関数にそれぞれ型を与えた方がよい。

## 関数適用時

デフォルト引数を持つ関数の適用だけが異なる挙動となる。

```Sign

` 通常は部分適用となる動作
f : [+] _ _ 3 4 5
f 1 2

g :
		x
		y : x + 1
		z : y + 1
	? x y z

` g のデフォルト引数は以下の条件で呼び出し可能

` 呼び出しの明示には .. を使う
g 3 .. ..

` 暗黙の呼び出しは、 if then elseと相当の動作となる。
` 例えば、予期せぬ演算で _ が算出された場合、以降の関数は _ に短絡評価可能

f 3 < 2 1

` デフォルト引数を持つ場合は、

g 1 3 < 2

` という適用が、来ても、暗黙でデフォルト引数での評価を行うこととする。
```
