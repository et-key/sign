# Sign言語のコメント規則 (改訂版)

## 基本原則

Sign言語におけるコメントは、行頭から始まる文字列リテラルと同義です。
よって、行頭に(`` ` ``)が存在すれば、それはコメントになるため、コメント文字列を閉じるための(`` ` ``)は、必要ありません。

なお、もう一つのハックですが、未定義の識別子の羅列は、Unitへ還元されるため、それを利用してもよいでしょう。

```sign
x : 1
```



## ルール定義

行頭に ` が指定されていること。

```sign
calc_func : x ?
`	これは関数のドキュメント（コメント）として機能する`
	`これは文字列リテラル、かつ後続のAtomは文字列として結合される。`
	x * 2
```
-> 上記の文字列は評価されますが、捨てられるためコメントとして機能します。

### B. トップレベル
トップレベルも1つの大きなブロック（またはモジュール定義）とみなせます。

```sign
`モジュールの説明
x : 1
```

-> この文字列も、`x` の定義に影響を与えないため、ドキュメントとして機能します。
（ただし、REPL環境などでは評価結果が表示される可能性があります）

### C. 具体例

```sign
	`これはコメントではない`
	`これは上のAtomと文字列結合される。`
`	これならコメントである
`	これは上のAtomと文字列結合`
	`最終行は前行までのAtomが文字列結合がされた返値になる`

```

## 文法仕様（PEG参照）

- 現状のpegでは、すべて `Comment` としてパースされます
- 字句解析レベルでコメントを削除しても構いません
- シンタックスハイライトやLanguage Serverは、**「行頭に ` が指定されている場合」** をコメント色で表示することが推奨されます
