# 新生Sign WASMコンパイラ Ph1 実装計画 (電卓レベル)

## 1. Ph1の目標範囲
複雑な制御ブロック、関数合成、リスト、メモリ操作（RAMマシン的挙動）には踏み込まず、WASMコンパイラの**「純粋なスタック計算基盤（パイプライン、NaN-Boxing、論理モデル）」**を確立することを目的とします。
* 対応機能: 数値・Registerリテラル、四則演算、比較演算、論理演算（ビット演算含む）、単一スコープ内での単純な変数定義、式内部のガード（三項演算子）。

## 2. 実装コンポーネント

### A. 基盤構造の作成
* ASTを受け取り、正規化 → スコープ解析 → WAT生成 を行う新しいパイプラインのスケルトンを作成。

### B. 型と演算の基礎実装 (WAT Generator)
* **NaN-Boxing**: リテラルはすべて `f64.const` として出力。
* **Register型とビット演算**: `0b...`, `0r...` を解析し、ビット演算子（`||`, `;;`, `&&`, `<<`, `>>`）適用時は、スタックの `f64` をインラインで `i32` にキャストして演算し、ふたたび `f64` に戻す命令を生成。
* **安全な除算**: `/` 実行時、ゼロ除算回避の基本的なWASM命令マッピング。

### C. 真偽値と論理崩壊の防止 (Ph1の最重要課題)
* **`$is_truthy` ヘルパー**: モジュール先頭に定義。スタックトップが `NaN` であれば `0`、それ以外なら `1` を返す `i32` のフラグ生成関数。
* **比較演算 (`<`, `==` 等)**: WASMのネイティブ比較直後に、結果が `0` なら `f64.const NaN` (Unit) を返し、`1` なら左辺(または右辺)の値を返すラップアセンブリを出力。

### D. 変数とローカルレジスタ
* Ph1ではエスケープ解析（Ph5の領域）を考慮しないため、パースされた変数は**すべて無条件でWASMの `local` 変数として宣言・使用**します。シャドウスタックは使用しません。

### E. 式内部のガード `:` と短絡評価 `&`, `|`
* `条件 : 真の値` などの式中に出現する三項演算子的なガードは、`$is_truthy` で評価し、WASMの `if (result f64) ... else f64.const NaN end` 構文にマッピングします。

## 3. Ph1の完了基準（テストケース）
以下のSignコードが正しくWATに変換され、実行時に期待通りの値を出力すること。

1. **算術とビット演算**:
   `10 + 2 * 3` -> `16`
   `0b1010 && 0b1100` -> `8`
2. **論理フォールバックと比較**:
   `10 < 5` -> `NaN` (Unit)
   `10 < 20 : 99` -> `99`
3. **変数定義（ローカルレジスタ稼働）**:
   `a : 100 \n b : a + 50 \n b` -> `150`
