# Phase 10: Zero Cost Domain Abstraction の具現化 (Range, Fold, Map)

## 目標
Signの最大の特徴である「Zero Cost Domain Abstraction（ゼロコストドメイン抽象化）」をWASM上で完全に具現化する。
巨大なリストを即座にメモリに展開するのではなく、軽量なイテレータとして保持し、`fold` や `map` が適用された瞬間に、関数呼び出しのオーバーヘッドを完全に排除した「WASMネイティブの高速ループ」へとコンパイル時（AST最適化時）に静的ディスパッチ（インライン展開）する。

## 実装内容

### 1. Range オブジェクトの遅延評価と軽量化
* `~` (Range) および各種ステップ演算子 (`~+`, `~-`, `~*`, `~/`, `~^`) を実装。
* リストをメモリ上に即時構築するのではなく、「開始値」「終了値」「ステップ値」「演算タイプ」の4つの情報のみを保持する32バイトの軽量構造体（Rangeイテレータ）をヒープに確保する。
* NaN-boxingの新しいタグとして以下を割り当て。
  * `0x7FF3...` (RangeStep): `0 ~+ 2` などの構築途中の中間状態。
  * `0x7FF2...` (Range): `0 ~+ 2 ~ 10` や `0 ~ 10` で完成したイテレータ状態。
* `start > end` の場合の降順リスト（負のステップ）を自動検知するロジックを実装。

### 2. インライン Fold の静的ディスパッチ
* `[+] [1 ~ 10]` や `[*] [1, 2, 3, 4, 5]` のような単一演算子による畳み込みを、クロージャの関数呼び出しを伴わずに計算する仕組みを実装。
* **AST最適化 (`optimizeAst`)**: パーサが生成した Operator Section の巨大なラムダ構造を静的にパターンマッチし、`inline_fold` ノードにすり替える (`extractSectionOp`)。
* **WASMアセンブリ生成 (`generateInlineFold`)**: 対象が Range(0x7FF2) の場合はメモリを消費しない数値ループへ、List(0x7FFC) の場合はポインタ走査ループへ直接展開する。ステップの正負による終了条件の動的切り替えも実装。

### 3. インライン Map の静的ディスパッチ
* `[* 2,] [1 ~ 5]` や `[x ? x * 2,] [1, 2, 3]` のように、末尾に `,` (リスト再構築バインダ) を伴う適用を Map 操作としてインライン展開する仕組みを実装。
* **AST最適化 (`optimizeAst`)**: ラムダや Operator Section の末尾が `,` と `unit` で終わっている構造を検知し、`inline_map` ノードにすり替える (`extractMapFunc`)。
* **WASMアセンブリ生成 (`generateInlineMap`)**: ループ内で直接式を評価し、同時にヒープメモリを確保して新しいコンスセルのチェイン（新しいリスト）を構築するアセンブリを出力。引数変数（`x` など）の参照をWASMの `$cur_val` レジスタへ紐付ける環境マップのハックを導入。

### 4. ASTの「巻き込み救済（Rescue）」
* 演算子の結合優先順位により、`[+] [1 ~ 10]` が `([+] 1 ~) 10` のように誤って解釈されてしまう問題を解決。
* `inline_fold` や `inline_map` の引数に未完成の Range (`1~` など) が巻き込まれていた場合、残りの引数と結合させて正しい Range オブジェクトを再構築してから Fold/Map に渡し直す Rescue 処理を `optimizeAst` に追加。

### 5. 開発環境 (Playground) の改善
* `index.html` の出力エリア (`Generated WAT` および `Execution Output`) を `<textarea readonly>` に変更し、Ctrl+A による全選択・コピーを容易にした。
