# 実装計画: WASM フェーズ 7〜9 (コンパイラ最適化・数学演算子・動的リスト構築)

## 目標
フェーズ6で「どんなコードでも正しく動く安全な動的ディスパッチ（最終防衛ライン）」が完成したことを受け、処理速度を極限まで高めるための「コンパイル時最適化（Optimizer）」を導入します。その後、高度な数学演算子や動的リスト構築の実装に進みます。

## ✅ フェーズ 7: コンパイル時最適化 (静的ディスパッチとβ簡約)
設計思想である「アセンブリコードが長くなっても処理速度を優先する」方針に従い、WASM/AArch64の実行時オーバーヘッドを削ぎ落としました。
- **定数畳み込み (Constant Folding):** JS側で計算可能な式をコンパイル時に事前評価し、最速のネイティブ定数ロードに置換。
- **β簡約 (Beta Reduction):** `(x ? x + 10) 20` のような即時関数を検知し、関数本体をインライン展開。
- **静的ディスパッチ:** `100 200` などをコンパイル時に検知し、動的ディスパッチをスキップして直接リスト構築命令を出力。
- **タプルのスプレッド展開:** `f (10 20)` を `(f 10) 20` のように自動でフラットに展開する仕様を確立。

## ✅ フェーズ 8: 単項数学演算子の完備
- **絶対値 `|...|`**: WASMネイティブの `f64.abs` に直接バインディング。
- **前置マイナス `-`**: 単項用の `f64.neg` を使用するように最適化。
- **階乗演算子 (後置 `!`)**: WASM内部で `loop` と `block` 命令を用いた高速なネイティブ階乗ルーチンを出力。NaN入力時の無限ループ防止機構（NaNガード）を実装。

## ✅ フェーズ 9: 動的リスト構築 (Range / Lift)
- **Range演算子 (中置 `~`)**: `1 ~ 5` のような動的リストをWASMヒープ上に構築する `$make_range` システムコールを実装。降順（`5 ~ 1`）や浮動小数点（`1.5 ~ 4.0`）にも対応する数学的ロジックを搭載し、末尾からの逆順構築によりアロケーションを極限まで高速化。
- **Lift演算子 (前置 `~`)**: `~42` を `42 , ()` にASTレベルで展開（Desugaring）することで、値を単一要素のリストに持ち上げる機能を実装。後置 `~` (Spread) と完全な双対（逆関数）を成す設計を実現。
