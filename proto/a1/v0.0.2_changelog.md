# v0.0.2 変更履歴

## バグ修正
- [x] **`0` (ゼロ) 消滅バグの修正**: `0 < x` などで `0` が正しくパースされるように修正しました。
- [x] **リテラル対応**: 
    - Hex (`0xFF`), Octal (`0o77`), Binary (`0b10`) を数値変換せず、文字列リテラルとしてそのまま出力するように変更しました。
    - エスケープ文字 (`\A` 等) が正しく保持されるようにスキャナを修正しました。
- [x] **演算子ロジックの改善**:
    - `~` (チルダ) の前置/中置判定を厳格化し、前置の場合は `~_` として出力するように変更しました（中置との区別のため）。
    - `head : x ~_` における `~_` (Restトークン) が `~` と `_` に分離されてしまう問題を修正しました。
    - 辞書定義リスト (`skills : a, b`) におけるカンマ処理を修正しました。
- [x] **定義のパース不具合の修正 (`concat_test` 等)**:
    - トークナイザが、特定の記号 (`:` や `[`) の前後に適切なスペーストークンを挿入していない問題を修正しました。これにより、`concat_test : [1 2]` のような定義が正しく認識されるようになりました。
    - 定義に使われる `:` を中置演算子としてより確実に判定するため、`tightOps` に `:` を追加しました。
- [x] **`main` リスト内の定義消失の修正**:
    - インデントされたバッククォート行（例: `	`基本リテラル`）が、単一行文字列として正しく認識されず、後続のコードを飲み込んでいた問題を修正しました（パーサ側で無効な文字列をコメントとしてスキップする処理を追加）。
